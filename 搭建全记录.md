## 一、起点：我想要什么

三个核心需求：

1. **一个微信 AI 伴侣** — 通过微信就能聊天的 AI，有记忆、有身份、有性格
2. **一个个人博客** — React 写的论坛式博客，有日记/相册/收藏/动漫板块
3. **一个管理界面** — 不用 SSH 敲命令就能管理 AI 的记忆、身份、配置

三个东西跑在同一台服务器上，互不干扰。

## 二、选型决策

### 服务器

| 选项 | 结论 |
|------|------|
| 腾讯云轻量应用服务器 | ✅ 4核4G 3Mbps 40G SSD，¥79/年新人价 |
| 系统选 OpenCloudOS 9.4 | RHEL 系，用 dnf 包管理 |
| 地区选上海 | 离用户近，延迟低 |

### AI 框架

| 选项 | 结论 |
|------|------|
| **OpenClaw** | ✅ Node.js 生态，原生支持微信插件，记忆系统完善 |
| 自研 | ❌ 轮子太大，微信协议、记忆系统、会话管理都要从头写 |

OpenClaw 的核心优势：
- 插件系统直接接入微信（`@tencent-weixin/openclaw-weixin`）
- 内置记忆系统（memory-core）：SQLite + FTS5 全文搜索 + 时间衰减 + MMR 多样性
- 内置 dreaming（梦境整合）：每天凌晨 3 点自动将分散记忆提炼到 DREAMS.md
- 内置 compaction：对话接近上下文上限时自动压缩旧消息为摘要
- 身份文件系统：IDENTITY.md / USER.md / SOUL.md / MEMORY.md

### 大模型

| 选项 | 结论 |
|------|------|
| **DeepSeek V4 Pro** | ✅ 1M 上下文，推理能力强，中文优秀，价格适中 |
| DeepSeek V4 Flash | 用于 compaction 和 dreaming，便宜 10 倍 |

关键认知：DeepSeek **没有 embedding API**，所以记忆搜索只能用纯文本模式（FTS5），不能用向量搜索。对个人聊天助手来说完全够用。

### 博客技术栈

| 层 | 选型 | 理由 |
|----|------|------|
| 前端 | React 19 + TypeScript + Vite 8 + Tailwind CSS 4 | 最新最热 |
| 后端 | Express 5 + SQL.js | 轻量，不需要独立数据库进程 |
| 数据库 | SQL.js (SQLite 编译到 WebAssembly) | 零配置，数据存在 JSON 文件中随项目走 |
| 状态管理 | Zustand | 比 Redux 轻 10 倍 |
| 设计风格 | Win98/Y2K 复古像素风 | 个性化，区别于千篇一律的扁平设计 |

## 三、阶段一：OpenClaw AI 伴侣诞生

### 3.1 安装与配置

```bash
npm install -g openclaw@latest
```

核心配置文件 `~/.openclaw/openclaw.json`（约 90 行），关键决策：

```
models.providers.deepseek        → 配置 API Key 和模型参数
gateway.mode = "local"           → 本地模式，不依赖 OpenClaw 云服务
gateway.bind = "loopback"        → 仅本机访问，不暴露公网
session.dmScope = "per-channel-peer" → 每个联系人独立上下文
memory.query.hybrid.vectorWeight = 0  → 纯文本搜索（DeepSeek 无 embedding）
plugins.entries.memory-core      → 启用记忆引擎
plugins.entries.active-memory    → 每次回复前自动检索记忆
```

### 3.2 微信接入

```bash
openclaw plugins install "@tencent-weixin/openclaw-weixin"
openclaw channels login --channel openclaw-weixin
```

终端打印二维码 → 手机微信扫码 → Bot 创建完成。微信 Bot 是**私人通道**，其他人搜不到。

### 3.3 记忆系统架构

```
每次对话
  ↓
active-memory 插件：检索相关记忆 → 注入 prompt
  ↓
对话结束：自动写入 memory/YYYY-MM-DD.md
  ↓
每天凌晨 3:00 dreaming：整合分散记忆 → DREAMS.md
  ↓
定期 heartbeat：提炼重要信息 → MEMORY.md
```

搜索方案：FTS5 全文搜索 + MMR 多样性去重 + 30天半衰期时间衰减

### 3.4 身份 Bootstrap

新部署的 AI 处于"金鱼记忆"状态，必须完成 Bootstrap：

| 文件 | 用途 | 内容 |
|------|------|------|
| IDENTITY.md | AI 身份 | 名字、性格、Emoji、自我认知 |
| USER.md | 用户信息 | 称呼、时区、背景 |
| SOUL.md | 行为准则 | 什么该做、什么不该做 |
| MEMORY.md | 长期记忆 | 精选的重要信息 |
| memory/YYYY-MM-DD.md | 每日记忆 | 自动积累 |

完成后删除 BOOTSTRAP.md，AI 就有了"灵魂"。

### 3.5 踩坑：systemd 重启循环

使用 `openclaw gateway install` 创建 systemd 服务后，Gateway 反复重启。

**原因**：systemd `Restart=always` + Gateway 自身退出码 78（端口被占用时的锁超时），形成无限重启循环。

**解决**：放弃 systemd，改用 crontab `@reboot`：

```bash
(crontab -l 2>/dev/null; echo '@reboot sleep 10 && nohup openclaw gateway run --port 18789 > /tmp/openclaw/gateway.log 2>&1 &') | crontab -
```

### 3.6 踩坑：Gateway stop 杀不死进程

`openclaw gateway stop` 只是停止服务注册，**不杀进程**。

**正确做法**：
```bash
kill -9 $(pgrep -f "openclaw gateway")
```

## 四、阶段二：博客与 AI 共存

### 4.1 共存架构

同一台服务器，三个服务各占端口，用 Nginx 做统一入口：

```
                    腾讯云轻量服务器 (4核4G 3Mbps)
                    ┌─────────────────────────────────┐
用户浏览器            │                                 │
────────→ :80       │  Nginx (systemd)                │
                    │  ├── /          → dist/ (静态)  │
                    │  ├── /api/*     → 127.0.0.1:3001│
                    │  └── /assets/*  → 30天缓存      │
                    │                                 │
用户微信              │  OpenClaw Gateway (crontab)     │
────────→ WeChat    │  └── 127.0.0.1:18789           │
                    │                                 │
                    │  PM2: myblog-api (Express)       │
                    │  数据: /opt/myblog/data/          │
                    └─────────────────────────────────┘
```

资源占用：博客 Express ~70MB + Nginx ~20MB + OpenClaw ~200MB ≈ **300MB / 4GB (7.5%)**

### 4.2 博客功能模块

| 页面 | 功能 |
|------|------|
| 首页 | 动态广场 + 统计 |
| 日记 | Markdown 发帖、评论、多板块 |
| 相册 | 照片上传、瀑布流、灯箱 |
| 收藏 | 网址收藏夹、分类、拖拽排序 |
| 动漫 | 动漫目录、评分、评论、追番 |
| 用户 | 个人主页、头像、签名 |
| AI 聊天 | 右下角浮动聊天窗口，接入 DeepSeek |

### 4.3 设计语言：Win98 复古风

设计决策：不做"现代扁平风"，做成 **Windows 98/2000 复古像素风格**。

关键元素：
- 3D 凸起按钮 (`win-btn`)：`border-color: #FFFFFF #808080 #808080 #FFFFFF`
- 蓝色渐变标题栏 (`win-titlebar`)：`linear-gradient(90deg, #000080, #1084D0)`
- 凹入输入框 (`win-field`)：`border: 2px inset #808080`
- 底色 `#ECE9D8` (经典 Win98 窗口灰)
- 主色调 `#FB7299` (B站粉)
- 像素字体 `Zpix` / `MS PGothic`
- 扫描线 + CRT 效果

### 4.4 认证系统

- bcrypt 密码哈希
- 自定义 session（基于 SQLite，不需要 Redis）
- Cookie: `connect.sid`，7 天过期
- 9 人注册上限（私人论坛）
- 管理员权限分级

### 4.5 部署流水线

```
本地 Windows:
  npm run build → dist/
  tar -czf → /tmp/myblog-deploy.tar.gz
  scp → root@124.221.10.157:/tmp/

服务器 Linux:
  tar -xzf → /opt/myblog/
  npm install --omit=dev
  pm2 restart myblog-api
  systemctl reload nginx
```

## 五、阶段三：可视化管理后台

### 5.1 动机

之前管理 AI 伴侣需要 SSH 到服务器手动操作：
- 编辑记忆文件要 `vim /root/.openclaw/workspace/memory/...`
- 修改身份要手动编辑 IDENTITY.md
- 看日志要 `tail -f /tmp/openclaw/gateway.log`
- 重启要记住那串 kill + nohup 命令

需要一个**可视化管理界面**，能通过浏览器完成所有操作。

### 5.2 设计原则

1. **融入博客** — 作为 `/admin/weixin` 路径，复用博客的认证和 UI 组件
2. **Win98 风格一致** — 不另起一套设计，保持复古风
3. **管理员专属** — 所有 API 使用 `requireAdmin` 中间件
4. **跨平台** — 开发环境 Windows，生产环境 Linux，自动适配路径

## 七、关键踩坑记录

### 坑 1：systemd 重启循环
- **现象**：Gateway 反复重启
- **原因**：`Restart=always` 遇退出码 78 触发无限循环
- **解决**：改用 crontab `@reboot`

### 坑 2：OpenCloudOS 不兼容 NodeSource
- **现象**：NodeSource setup 脚本报"不支持的系统"
- **原因**：OpenCloudOS 不被识别为 RPM 系
- **解决**：用 nvm 安装 Node.js 24

### 坑 3：DeepSeek 无 Embedding API
- **现象**：开启向量搜索后报错
- **原因**：DeepSeek 没有 embedding 模型
- **解决**：`vectorWeight: 0, textWeight: 1.0`，纯 FTS5 文本搜索

### 坑 4：Gateway stop 不杀进程
- **现象**：`openclaw gateway stop` 后端口仍被占用
- **原因**：stop 只停服务注册
- **解决**：`kill -9 $(pgrep -f "openclaw gateway")`

### 坑 5：SSH 非交互式不加载 nvm
- **现象**：SSH 执行命令提示 `openclaw: command not found`
- **原因**：非交互式 shell 不加载 `.bashrc`
- **解决**：脚本开头加 `export NVM_DIR` + `source nvm.sh`

### 坑 6：VITE_ 环境变量构建时硬编码
- **现象**：改 `.env` 后 AI 聊天仍然用旧 Key
- **原因**：`VITE_*` 变量在 `vite build` 时嵌入 JS bundle
- **解决**：改 `.env` 后必须重新 `npm run build`

### 坑 7：模型上下文窗口被限制到 200K
- **现象**：DeepSeek V4 Pro 应该有 1M 上下文，实际只有 200K
- **原因**：DeepSeek 插件默认限制
- **解决**：在 `models.providers.deepseek.models[]` 显式配置 `contextWindow: 1000000`

## 八、运维手册

### 日常检查

```bash
# 进入服务器
ssh root@124.221.10.157

# 所有服务状态
pm2 status                              # myblog-api: online
systemctl is-active nginx               # active
pgrep -f "openclaw gateway"             # 有 PID = 运行中

# 端口检查
ss -tlnp | grep -E "80|3001|18789"     # 三个端口都应 LISTEN

# 资源
free -h                                 # 内存
df -h                                   # 磁盘
```

### 重启各服务

```bash
# 博客 API
pm2 restart myblog-api

# Nginx
systemctl reload nginx       # 重载配置（不中断）
systemctl restart nginx      # 完全重启

# OpenClaw Gateway
kill -9 $(pgrep -f "openclaw gateway")
nohup /root/.nvm/versions/node/v24.16.0/bin/openclaw gateway run --port 18789 > /tmp/openclaw/gateway.log 2>&1 &
```

### 更新博客/管理后台

```bash
# 本地
cd ~/Desktop/myblog
npm run build
tar -czf /tmp/myblog-deploy.tar.gz --exclude='node_modules' --exclude='src' dist/ server/ package.json package-lock.json
scp -i ~/.ssh/id_ed25519 /tmp/myblog-deploy.tar.gz root@124.221.10.157:/tmp/

# 服务器
ssh root@124.221.10.157
export NVM_DIR="$HOME/.nvm" && [ -s "$NVM_DIR/nvm.sh" ] && . "$NVM_DIR/nvm.sh"
cd /opt/myblog
tar -xzf /tmp/myblog-deploy.tar.gz && rm /tmp/myblog-deploy.tar.gz
npm install --omit=dev
pm2 restart myblog-api
```
